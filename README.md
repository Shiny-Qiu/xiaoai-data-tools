# 小爱例行数据整理网页工具

基于 Django 的本地网页工具。支持三大功能：**Session 整理**、**Topic 废除判定**、**人力分配**。

> 本工具仅供本地开发使用（DEBUG=True，内置开发密钥），不适合部署到公网或生产环境。

## 功能概览

工具提供三个独立的功能页签，覆盖标注数据处理的主要环节：

### Session 整理

对原始 session 数据做列整理、排序和拆分。

- 支持自定义目标列顺序（一行一个字段名，也兼容 Python 列表格式）
- 按 `add_session_id` 和 `timestamp` 排序
- 支持自定义拆分行数（默认 500 行）
- 缺失列处理：严格模式（缺列即失败） / 宽松模式（缺失普通列补空）
- 即使是宽松模式，排序必需字段 `add_session_id` 和 `timestamp` 也不允许缺失

### Topic 废除判定

根据日期、query 有效性、Agent 类型和 topic 类型规则，判定哪些 topic 应保留、哪些应废除。

- 支持自定义目标日期（格式 `YYYY-MM-DD`）
- 支持自定义 RETAIN_RULES 保留规则（格式：`Agent,topic类型,数量`，每行一条）
- 支持自定义跨垂域 topic 保留数量
- 支持选择候选 topic 保留顺序：按行数升序 / 随机顺序 / topic 排序 / 原始文档顺序
- 输出文件新增 `是否废除` 列，命中保留规则的标记为 `否`，其余为 `是`

### 人力分配

将保留的 topic 按规则分配给标注员。

- 支持自定义规范类别时间配置（格式：`类别名,分钟数`）
- 支持自定义标注员配置（格式：`标注员名:类别1,类别2,...`）
- 支持自定义每人目标分钟数（默认 300 分钟）
- 仅处理 `是否废除` 为 `否` 的数据
- 分配单位是整个 topic，同一 topic 内所有类别必须全部被标注员覆盖才能分配
- 输出文件新增 `理想态标注员` 列

## 通用特性

- 支持上传 `.csv`、`.xls`、`.xlsx` 文件
- 支持一次上传多个文件批量处理
- 后台异步处理，页面实时显示进度
- 输出统一为 `.xlsx` 格式，多个结果文件打包为 ZIP 下载
- 处理完成后展示结果摘要（成功/失败文件数、警告信息等）

## 处理流程

```text
Session 整理：上传文件 → 校验列名/模式 → 按目标列整理 → 排序 → 按行数拆分 → 导出 Excel → 打包 ZIP 下载
Topic 废除：上传文件 → 校验必需字段 → 按目标日期筛选 → 剔除全不参评topic → 按规则保留 → 写入是否废除 → 打包 ZIP 下载
人力分配：  上传文件 → 校验必需字段 → 筛选未废除数据 → 计算topic时长 → 贪心分配标注员 → 写入理想态标注员 → 打包 ZIP 下载
```

## 输入与输出

### Session 整理

- 输入：包含 `add_session_id`、`timestamp` 及目标列的表格文件
- 输出：`原文件名_整理后.xlsx`（单文件时）或 `原文件名_整理后_序号.xlsx`（多文件时）

### Topic 废除

- 输入：包含 `真实细分Agent`、`topic_id`、`topic类型`、`标注日期`、`query有效性` 的表格文件
- 输出：`原文件名_处理结果.xlsx`

### 人力分配

- 输入：包含 `是否废除`、`topic_id`、`规范类别` 的表格文件
- 输出：`原文件名_人力分配结果.xlsx`

## 快速开始

前置条件：本机已安装 [Python 3.13+](https://www.python.org/downloads/)（项目依赖 Django 6，要求 Python 3.13 或以上）。

**所有依赖已随项目一起打包在 `packages/` 目录中，无需联网下载。**

### Windows（双击即用）

双击项目根目录下的 `start.bat`。

脚本会自动完成：创建虚拟环境 → 从本地 `packages/` 安装依赖 → 启动服务 → 打开浏览器。

也可以在终端中执行：

```powershell
.\start.bat
```

### macOS / Linux

```bash
./start.sh
```

启动成功后浏览器会自动打开 `http://127.0.0.1:8000`，页面顶部有三个功能页签可切换。

> 本地离线包为 Windows + Python 3.13 编译版本。如果你使用其他系统或 Python 版本，启动脚本会自动回退到从网络安装依赖。

## 页面使用步骤

### Session 整理

1. 点击「Session整理」页签
2. 上传一个或多个 `.csv` / `.xls` / `.xlsx` 文件
3. 设置目标列顺序（默认已填充，可按需调整）
4. 设置拆分行数
5. 选择缺失列处理方式
6. 点击「开始处理」
7. 等待进度完成后下载 ZIP 结果包

### Topic 废除

1. 点击「Topic废除」页签
2. 上传包含必需字段的文件
3. 设置目标日期
4. 选择候选 topic 保留顺序
5. 设置跨垂域 topic 保留数量
6. 按需调整 RETAIN_RULES
7. 点击「开始处理」
8. 查看处理日志（包含每条规则的匹配情况）并下载结果

### 人力分配

1. 点击「人力分配」页签
2. 上传包含 `是否废除` 等必需字段的文件
3. 按需调整规范类别时间配置
4. 按需调整标注员配置
5. 设置每人目标分钟数
6. 点击「开始分配」
7. 查看分配详情并下载结果

## 常见问题

### 双击 start.bat 闪退

通常是因为未安装 Python 3.13+。请先安装 [Python](https://www.python.org/downloads/)，安装时勾选「Add Python to PATH」。

### 页面打不开

确认终端已显示 `Starting development server at http://127.0.0.1:8000/`，并检查浏览器是否访问了正确地址。

### 8000 端口被占用

改用其他端口：

```bash
python manage.py runserver 8001
```

然后访问 `http://127.0.0.1:8001`。

### Session 整理上传后报错

检查输入文件是否包含 `add_session_id` 和 `timestamp` 两个排序必需字段，即使宽松模式也不能缺少。

### Topic 废除没有保留任何 topic

检查目标日期是否与文件中的 `标注日期` 匹配，以及 RETAIN_RULES 中的 Agent 和 topic 类型是否与数据一致。

### 人力分配提示没有有效数据

确认输入文件中存在 `是否废除` 为 `否` 的记录，且 `规范类别` 在类别时间配置中有对应条目。
